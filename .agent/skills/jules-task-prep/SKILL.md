# Подготовка задач для Jules

## Обзор
Скилл для подготовки задач в формате, оптимальном для Jules (Google AI agent). Генерирует Markdown-описание задачи, которое можно скопировать в Jules для асинхронной ночной работы.

## Когда использовать
- Пользователь хочет делегировать задачи Jules на ночь
- Нужно подготовить batch из задач (SQL, n8n JSON, фронтенд)
- Пользователь говорит: «подготовь задачу для Jules», «кинь в Jules»

## Типы задач

### 1. SQL-задача
Jules генерирует SQL (миграции, запросы). Утром Antigravity проверяет через `EXPLAIN` и выполняет.

### 2. n8n-задача
Jules генерирует workflow JSON. Утром Antigravity валидирует через MCP и деплоит.

### 3. Фронт-задача
Jules создаёт/правит React-компоненты. Утром merge + интеграция с API.

## Формат задачи для Jules

Каждая задача ДОЛЖНА содержать эти секции:

```markdown
# [Чёткий заголовок задачи]

## Контекст проекта
Прочитай файлы:
- `docs/jules/project-context.md` — стек и архитектура
- `docs/jules/[db-schema|n8n-reference|frontend-patterns].md` — релевантный reference

## Что нужно сделать
1. [Конкретный шаг 1]
2. [Конкретный шаг 2]
3. ...

## Файлы для изменения
- `path/to/file.ts` — что именно менять
- `path/to/new-file.ts` [NEW] — новый файл

## Acceptance Criteria
- [ ] [Критерий 1]
- [ ] [Критерий 2]

## Ограничения
- НЕ трогать: [список файлов]
- НЕ менять: [API контракты/типы/etc]
```

## Процесс подготовки (ОБЯЗАТЕЛЬНЫЙ)

> ⚠️ **НИКОГДА не отправлять задачи напрямую через API без согласования!**

### Шаг 1: Подготовка файлов задач
1. **Определи тип задачи** (SQL / n8n / фронт / смешанная)
2. **Собери контекст** — какие файлы/таблицы/workflow затронуты
3. **Создай файл задачи** в `docs/jules/tasks/XX-название.md` в формате выше
4. **Укажи reference-доки** — какие `docs/jules/*.md` прочитать
5. **Добавь acceptance criteria** — как проверить результат
6. **Добавь ограничения** — что НЕ трогать

### Шаг 2: Согласование с пользователем
**ПЕРЕД** отправкой через API — ОБЯЗАТЕЛЬНО:
1. Показать пользователю список задач с кратким описанием
2. Для каждой задачи указать: какие файлы затрагиваются
3. Если несколько задач — провести **анализ конфликтов** (см. ниже)
4. Получить явное **подтверждение** от пользователя: «отправляй»

### Шаг 3: Отправка
1. Запушить файлы задач в репо (`git push`)
2. Отправить через API с коротким prompt: «Прочитай и выполни docs/jules/tasks/XX-название.md»
3. Использовать `automationMode: AUTO_CREATE_PR`

## Правила

### Изолированность
Каждая задача должна быть **самодостаточной**. Jules не имеет контекста предыдущих задач.

### Конкретность
- ❌ «Сделай фичу поиска» — слишком абстрактно
- ✅ «Добавь компонент SearchBar.tsx в components/ с input, debounce 300ms, callback onSearch(query: string)»

### Reference-файлы
Всегда указывай какие `docs/jules/*.md` нужно прочитать. Без контекста Jules будет галлюцинировать.

### Одна задача — один PR
Не объединяй несвязанные изменения в одну задачу.

### 🚨 Правила параллельных задач (КРИТИЧНО)

При отправке нескольких задач одновременно:

1. **Анализ конфликтов файлов** — ОБЯЗАТЕЛЬНО перед отправкой батча:
   - Составь таблицу: задача → какие файлы трогает (✏️ edit / 🆕 new)
   - Если **2+ задачи редактируют ОДИН файл** → это КОНФЛИКТ
   - Показать пользователю таблицу конфликтов

2. **Параллельно МОЖНО** — задачи на **разные файлы**:
   - ✅ Задача A создаёт `utils/foo.ts` + Задача B создаёт `components/Bar.tsx`
   - ✅ SQL миграции в разных файлах
   - ✅ Новые n8n workflow в разных JSON

3. **Параллельно НЕЛЬЗЯ** — задачи на **один файл**:
   - ❌ Задача A рефакторит `App.tsx` + Задача B добавляет фичу в `App.tsx`
   - ❌ Две задачи меняют `types.ts`
   - Решение: объединить в одну задачу ИЛИ отправить последовательно

4. **Порядок мержа** — при наличии рефакторинга:
   - Рефакторинг мержить ПЕРВЫМ
   - Затем остальные задачи (могут потребовать ручной адаптации)

## Jules Environment Setup

Для настройки окружения Jules используй setup script:

```bash
npm install --ignore-scripts
git checkout -- . || true
git clean -fd || true
```

Это устанавливает зависимости без postinstall и очищает рабочее дерево.

## Отправка через API

Prompt для Jules должен быть **коротким** — ссылаться на файл задачи:

```
Прочитай и выполни задачу из файла docs/jules/tasks/04-carousel-editor.md.
Также прочитай docs/jules/project-context.md для контекста проекта.
```

НЕ дублировать содержимое файла в prompt — Jules прочитает файл из репо.

## Подготовка reference-документов для нового проекта

При добавлении нового проекта в Jules необходимо создать `docs/jules/` с файлами:

1. `project-context.md` — стек, архитектура, структура каталогов, ключевые типы
2. `db-schema.md` — таблицы, типы колонок, примеры SQL
3. `n8n-reference.md` — паттерны workflow, JSON примеры нод, endpoints
4. `frontend-patterns.md` — компоненты, API, дизайн-система, конвенции

Antigravity может сгенерировать эти документы автоматически — просто попроси: «подготовь reference-доки для Jules».
