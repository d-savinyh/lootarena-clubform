# Тесты 6-10: Количественная валидация

Детальные промпты и инструкции для тестов 6-10. Эти тесты фокусируются на **количественном анализе**: рынок, конкуренция, экономика, монетизация, фаундер.

---

## Тест 6: Market Sizing (TAM/SAM/SOM)

### Суть
Есть ли деньги в этой нише? Рынок $100M — это хорошо для micro-SaaS. Рынок $1M — мало.

### Как проводить

Оцени три уровня:

1. **TAM** (Total Addressable Market) — весь теоретический рынок
   - Сколько людей/компаний В МИРЕ имеют эту проблему?
   - Сколько они тратят на решение (в год)?
   - TAM = количество × средний чек × 12

2. **SAM** (Serviceable Addressable Market) — доступный рынок
   - Фильтр по географии, языку, сегменту
   - Кого ты реально можешь обслужить?

3. **SOM** (Serviceable Obtainable Market) — реалистичная доля
   - Сколько клиентов ты можешь получить за первый год?
   - С учётом маркетинг-бюджета, команды, конкурентов

Также оцени:
- **Тренд рынка:** растёт / стабильный / падает?
- **CAGR** (среднегодовой темп роста) если доступен

### Критерии оценки

| Балл | Критерий |
|------|----------|
| 9-10 | TAM > $10B, SAM > $100M, растёт > 20%/год. Micro-SaaS может занять долю |
| 7-8 | TAM $1-10B, SAM $10-100M, растёт 10-20%/год |
| 5-6 | TAM $100M-1B, SAM $5-10M, стабильный или медленный рост |
| 3-4 | TAM < $100M, SAM < $5M, рынок стагнирует |
| 1-2 | Рынок не существует или сжимается. Micro-SaaS не найдёт клиентов |

### Формат вывода

```

| Уровень | Размер | Как считали |
|---------|--------|-------------|
| TAM | $X | [Логика расчёта] |
| SAM | $X | [Фильтры] |
| SOM (1 год) | $X | [Реалистичная оценка] |

**Тренд:** [Растёт X%/год / Стабильно / Падает]
**Источники/аналогии:** [На что опирались]

## Тест 7: Competitive Moat

### Суть
Если идея хорошая — почему её ещё не сделали? А если сделали — почему ты победишь?

### Как проводить

1. **Прямые конкуренты** — кто делает ТОЧНО то же самое?
2. **Непрямые конкуренты** — кто решает ту же проблему иначе?
3. **"Сделай сам" конкурент** — может юзер решить проблему Excel-ом/Notion/скриптом?

Для каждого конкурента:
- Что у них хорошо?
- Что плохо (где дыра)?
- Сколько стоит?

Затем оцени **моат (защитный ров)** продукта:

| Тип моата | Применимо? |
|-----------|-----------|
| **Network Effects** — ценность растёт с числом юзеров | Да/Нет |
| **Data Moat** — уникальные данные, которых нет у конкурентов | Да/Нет |
| **Switching Costs** — юзеру больно переключиться | Да/Нет |
| **Brand/Community** — лояльное сообщество | Да/Нет |
| **Speed of Execution** — ты быстрее других | Да/Нет |

### Критерии оценки

| Балл | Критерий |
|------|----------|
| 9-10 | Нет прямых конкурентов + 2+ типа моата. Уникальная комбинация |
| 7-8 | 1-2 конкурента, но есть чёткая дифференциация + 1 моат |
| 5-6 | Конкуренты есть, дифференциация слабая. Можно скопировать за месяц |
| 3-4 | Сильные конкуренты, нет моата. «Ещё один X» |
| 1-2 | FAANG или VC-backed конкурент. Проблема решена лучше и дешевле |

### Формат вывода

```
### 7. Competitive Moat — X/10

**Прямые конкуренты:**
| Конкурент | Цена | Сила | Дыра |
|-----------|------|------|------|
| [Название] | $X/мес | [Что хорошо] | [Что плохо] |

**Непрямые конкуренты:** [Кто решает проблему иначе]
**"Сделай сам":** [Может ли юзер обойтись без продукта?]

**Моат:**
- [Тип] — [Есть/Нет, пояснение]

**Вердикт:** [Итог]

```

## Тест 8: Unit Economics

### Суть
Зарабатываешь ли ты больше с одного клиента, чем тратишь на его привлечение? Если нет — масштабирование только ускорит банкротство.

### Как проводить

Оцени ключевые метрики:

1. **ARPU** (Average Revenue Per User/Month) — сколько платит средний юзер?
2. **Churn Rate** (Monthly) — какой % юзеров уходит в месяц?
3. **LTV** (Lifetime Value) — ARPU / Churn = сколько юзер принесёт за время жизни
4. **CAC** (Customer Acquisition Cost) — сколько стоит привлечь 1 клиента?
5. **LTV/CAC ratio** — должно быть > 3 для здорового бизнеса

Также:
- **Маржа** — серверные/API расходы на 1 юзера
- **Payback period** — через сколько месяцев юзер "окупится"
- **Break-even** — сколько платящих клиентов нужно для окупаемости

### Критерии оценки

| Балл | Критерий |
|------|----------|
| 9-10 | LTV/CAC > 5. Маржа > 80%. Payback < 3 месяца. Юнит-экономика отличная |
| 7-8 | LTV/CAC 3-5. Маржа 60-80%. Payback 3-6 месяцев |
| 5-6 | LTV/CAC 1-3. Маржа 40-60%. На грани окупаемости |
| 3-4 | LTV/CAC < 1. Каждый юзер убыточен. Нужно менять модель |
| 1-2 | Экономика не сходится в принципе. Бесплатный продукт без монетизации |

### Формат вывода

```
### 8. Unit Economics — X/10

| Метрика | Значение | Комментарий |
|---------|----------|-------------|
| ARPU | $X/мес | [Как посчитали] |
| Churn | X%/мес | [Оценка] |
| LTV | $X | ARPU / Churn |
| CAC | $X | [Основные каналы] |
| LTV/CAC | X:1 | [Здоровый > 3:1] |
| Маржа | X% | [Расходы на юзера] |
| Break-even | X клиентов | [При каком MRR] |

## Тест 9: Willingness to Pay (WTP)

### Суть
Юзеры говорят «классная идея!» бесплатно. Но достать кредитку — совсем другое дело. Заплатят ли за это?

### Как проводить

Анализируй индикаторы WTP:

1. **Платят ли за аналоги?** Если конкуренты берут $X/мес и у них есть клиенты — WTP подтверждена
2. **Ценность в деньгах** — сколько юзер ЭКОНОМИТ/ЗАРАБАТЫВАЕТ благодаря продукту?
3. **Price Sensitivity** — целевая аудитория богатая или бедная?
4. **Freemium жизнеспособность** — можно ли дать бесплатно ядро, а монетизировать PRO?
5. **Replacement budget** — какой бюджет юзер уже тратит на текущее решение?

Определи **оптимальную цену**:
- Ниже конкурентов → быстрый рост, но низкая маржа
- На уровне конкурентов → нужна чёткая дифференциация
- Выше конкурентов → premium positioning, нужен wow-фактор

### Критерии оценки

| Балл | Критерий |
|------|----------|
| 9-10 | Аналоги стоят $X и люди платят. Продукт экономит > 10x своей цены. WTP очевидна |
| 7-8 | Есть платные аналоги. Ценность понятна. Цена адекватна рынку |
| 5-6 | Аналоги бесплатные, но есть ниша для premium. WTP не доказана, но возможна |
| 3-4 | Аудитория привыкла к бесплатному. Ценность трудно объяснить. Кредитку не достанут |
| 1-2 | Никто не платит за подобное. Ценность не генерирует деньги. Только ad-supported |

### Формат вывода

```
### 9. Willingness to Pay — X/10

**Платят ли за аналоги:** [Да/Нет, примеры с ценами]
**Ценность продукта в деньгах:** [Сколько экономит/зарабатывает юзер]
**Replacement budget:** [$X/мес — что юзер уже тратит]
**Оптимальная цена:** $X/мес — [Почему]
**Freemium модель:** [Работает / Не работает, почему]

**Вердикт:** [Итог]
**Балл: X/10** — [Почему]
```

## Тест 10: Founder-Market Fit

### Суть
Идею может убить не рынок, а несовпадение фаундера и ниши. Разработчик, который ненавидит маркетинг, не выживет в consumer-продукте. Дизайнер без технических навыков не потянет инфраструктурный SaaS.

### Как проводить

Оцени совпадение по 5 осям:

1. **Domain expertise** — разбираешься ли ты в проблемной нише?
   - Личный опыт боли (Dog-fooding potential)
   - Есть ли доступ к целевой аудитории?

2. **Tech fit** — твои навыки подходят для этого стека?
   - Можешь ли реалистично собрать MVP за 2-4 недели?
   - Нужны ли навыки, которых у тебя нет? (ML, mobile, specific API)

3. **Distribution fit** — умеешь ли ты дотянуться до клиентов?
   - Есть ли аудитория (Twitter, YouTube, блог)?
   - Есть ли сеть контактов в нише?
   - Готов ли ты к холодным продажам?

4. **Motivation fit** — будешь ли ты гореть этим через 6 месяцев?
   - Это проблема, которая тебя лично бесит?
   - Или это «объективно хорошая идея», которая тебе безразлична?

5. **Resource fit** — хватит ли ресурсов до первого дохода?
   - Финансовая подушка на 6-12 месяцев?
   - Время: full-time или side-project?

### Критерии оценки

| Балл | Критерий |
|------|----------|
| 9-10 | Dog-fooding (сам юзер). Есть аудитория. Стек знакомый. Горит темой |
| 7-8 | 3-4 из 5 осей совпадают. Недостающее можно восполнить за месяц |
| 5-6 | 2-3 совпадения. Придётся учиться на ходу. Рискованно, но возможно |
| 3-4 | 1 совпадение. Нет домейн-экспертизы, нет аудитории, стек незнаком |
| 1-2 | Полное несовпадение. Идея из чужого мира |

### Формат вывода

```
### 10. Founder-Market Fit — X/10

| Ось | Оценка | Пояснение |
|-----|--------|-----------|
| Domain expertise | 🟢/🟡/🔴 | [Есть ли опыт в нише] |
| Tech fit | 🟢/🟡/🔴 | [Подходит ли стек] |
| Distribution fit | 🟢/🟡/🔴 | [Есть ли доступ к клиентам] |
| Motivation fit | 🟢/🟡/🔴 | [Горишь ли этим] |
| Resource fit | 🟢/🟡/🔴 | [Хватит ли ресурсов] |

**Вердикт:** [Итог]
**Балл: X/10** — [Почему]
```
