# Чек-лист валидации Skill

Используй этот чек-лист для проверки skill перед и после загрузки.

---

## Перед началом разработки

- [ ] Определены 2-3 конкретных use cases с триггерами
- [ ] Определены инструменты (built-in или MCP)
- [ ] Изучены похожие skills для референса
- [ ] Спланирована структура папок
- [ ] Проверено: можно ли расширить существующий skill вместо создания нового

---

## Во время разработки

### Структура

- [ ] Папка названа в kebab-case
- [ ] Файл называется точно `SKILL.md` (регистр важен!)
- [ ] Нет `README.md` внутри папки skill
- [ ] SKILL.md < 5000 слов (детали вынесены в `references/`)

### YAML Frontmatter

- [ ] Начинается с `---`
- [ ] Заканчивается `---`
- [ ] Поле `name` в kebab-case, без пробелов, без заглавных
- [ ] Поле `name` совпадает с именем папки
- [ ] Поле `description` содержит ЧТО делает skill
- [ ] Поле `description` содержит КОГДА использовать (триггеры)
- [ ] Поле `description` < 1024 символов
- [ ] Нет символов `<` или `>` (XML запрещён)
- [ ] Нет префиксов `claude-` или `anthropic-` в name

### Инструкции

- [ ] Критические правила в секции `## ⚠️ Критические правила`
- [ ] Чёткие и actionable шаги (bullet points, нумерация)
- [ ] Обработка ошибок описана
- [ ] Секция `## Troubleshooting` присутствует
- [ ] Секция `## Примеры` присутствует
- [ ] Ссылки на `references/` корректны (файлы существуют)
- [ ] Неоднозначные формулировки заменены на конкретные чек-листы
- [ ] Критичные проверки вынесены в скрипты (`scripts/`)

---

## Перед загрузкой

### Triggering Tests

- [ ] Подготовлены 3+ запроса которые ДОЛЖНЫ триггерить skill
- [ ] Подготовлены 3+ запроса которые НЕ ДОЛЖНЫ триггерить skill
- [ ] Trigger rate > 90% на релевантных запросах
- [ ] Отладочная команда: «Когда бы ты использовал skill [name]?»

Шаблон:
```
Should trigger:
- "[Очевидная фраза 1]"
- "[Очевидная фраза 2]"
- "[Перефразированный запрос]"

Should NOT trigger:
- "[Нерелевантный запрос]"
- "[Запрос для другого skill]"
- "[Общий вопрос]"
```

### Functional Tests

- [ ] Подготовлены тест-кейсы в формате Given/When/Then
- [ ] Стандартные сценарии проходят
- [ ] Edge cases проверены (пустые данные, невалидный ввод)
- [ ] API вызовы успешны (0 ошибок)
- [ ] Интеграция с MCP инструментами работает

Шаблон:
```
Test: [Название сценария]
Given: [Начальное состояние]
When: [Действие пользователя]
Then:
  - [Ожидаемый результат 1]
  - [Ожидаемый результат 2]
  - Нет ошибок API
```

### Performance Comparison (Baseline)

- [ ] Замерены метрики без skill (tool calls, tokens, retry rate)
- [ ] Замерены метрики с skill
- [ ] Skill улучшает baseline

Шаблон:
```
Without skill: X messages, Y failed calls, Z tokens
With skill:    A messages, B failed calls, C tokens
```

### Финальная подготовка

- [ ] Skill сжат в .zip файл (если требуется для Claude.ai)
- [ ] Документация актуальна
- [ ] Автоматическая валидация пройдена

## После загрузки

- [ ] Протестирован в реальных диалогах (3-5 запросов)
- [ ] Мониторинг under-triggering (skill не загружается → добавить триггеры)
- [ ] Мониторинг over-triggering (skill загружается на всё → добавить негативные триггеры)
- [ ] Мониторинг execution issues (непоследовательные результаты → улучшить инструкции)
- [ ] Собрана обратная связь
- [ ] Edge cases записаны и внесены в skill
- [ ] Обновлена версия в metadata

## Quick Validation Commands

```bash
# Проверить имя папки
ls -la .agent/skills/ | grep -i {skill-name}

# Проверить наличие SKILL.md
ls .agent/skills/{skill-name}/SKILL.md

# Посчитать слова в SKILL.md
wc -w .agent/skills/{skill-name}/SKILL.md

# Запустить валидатор
python .agent/skills/skill-creator/scripts/validate_skill.py .agent/skills/{skill-name}

# Отладка триггеров
# Спроси Claude: "Когда бы ты использовал skill {skill-name}?"
```

## Распространённые ошибки

| Ошибка | Решение |
|--------|---------|
| "Could not find SKILL.md" | Переименовать файл в `SKILL.md` |
| "Invalid frontmatter" | Проверить `---` разделители и YAML синтаксис |
| "Invalid skill name" | Убрать пробелы и заглавные буквы из `name` |
| Skill не триггерится | Добавить конкретные фразы в `description` |
| Skill триггерится на всё | Добавить негативные триггеры в `description` |
| MCP вызовы падают | Проверить подключение, ключи, имена инструментов |
| Claude не следует инструкциям | Упростить, критическое — в начало, скрипты для проверок |
| Деградация качества | SKILL.md < 5000 слов, references/, не >50 skills |
