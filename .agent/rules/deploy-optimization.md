
Перед деплоем (любым: Dokploy, Docker, CI/CD) — **всегда проверяй `.dockerignore`**:

1. Если файла нет — создай его автоматически.
2. Минимальные обязательные исключения:
   ```
   .git
   node_modules
   dist
   ```
3. Дополнительно исключай по контексту проекта:
   - `.agent`, `.agents`, `.gemini`, `specs` — инфраструктура агента
   - Мобильные/другие подпроекты не участвующие в текущем билде
   - Тестовые данные, архивы, медиа-файлы (`*.zip`, `*.mp4`, `*.png` если не нужны)
   - Документация (`*.md`, `LICENSE`)
4. Проверяй размер Docker-контекста мысленно: если в корне есть директории > 50 МБ не нужные для сборки — добавь в `.dockerignore`.

Без `.dockerignore` Docker передаёт ВСЕ файлы в контекст сборки — это может замедлить деплой в 5-10 раз.
