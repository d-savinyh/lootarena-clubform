---
description: Правила безопасности — никаких разрушительных действий без подтверждения.
---

Ничего разрушительного (DROP/TRUNCATE/массовые DELETE, выключение workflow, перезапись секретов) не делать без явного подтверждения от пользователя.

Если задача затрагивает данные — предложи безопасный план (бэкап/транзакция/миграция/rollback).
