
## Запрещённые паттерны

Каждый раз когда агент допускает ошибку, которая уже случалась — добавь строку сюда.
Этот файл — главный предохранитель от повторных ошибок.

### n8n
- НЕ используй typeVersion: 1 для нод — всегда ставь latest доступную версию
- НЕ изменяй/удаляй workflow без явного подтверждения пользователя
- НЕ хардкодь URL n8n в workflows/скиллах — бери из `toolkit.json → project.n8n_url`

### Git
- НЕ делай `git commit` без предварительного `git diff --stat` + просмотра ключевых diff
- НЕ полагайся на память о сессии при формировании коммит-сообщения — ВСЕГДА читай `git diff`
- НЕ пуши без проверки `git status --short` после коммита

### Postgres
- НЕ делай DROP/TRUNCATE/массовый DELETE без подтверждения пользователя
- НЕ меняй схему без бэкапа или транзакции

### Общее
- НЕ хардкодь проектоспецифичные значения (URL, имена, пути) в universal-файлах toolkit
- НЕ давай пустых обещаний «в будущем буду делать X» — если важно, запиши в правило/скилл
- НЕ создавай файл без проверки что он уже существует
