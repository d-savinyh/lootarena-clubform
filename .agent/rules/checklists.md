
## Предполётные чеклисты

Перед выполнением определённых действий — пройди соответствующий чеклист.
Каждый обнаруженный баг → новый пункт в нужном чеклисте.

---

### Перед изменением n8n workflow
- [ ] Прочитай `/specs/n8n/workflows.md` — что за workflow, как работает
- [ ] Проверь `projects.json` — какие проекты используют этот n8n инстанс
- [ ] Сделай `n8n_get_workflow` перед изменением — зафиксируй текущее состояние
- [ ] После изменения — обнови `/specs/n8n/workflows.md`

### Перед изменением схемы Postgres
- [ ] Прочитай `/specs/db/schema.md`
- [ ] Предложи безопасный план (транзакция/бэкап)
- [ ] После изменения — обнови `/specs/db/schema.md`

### Перед деплоем
- [ ] Проверь `.dockerignore`
- [ ] `git status` — всё закоммичено?
- [ ] Просмотри diff с последнего тега
- [ ] Собери подробный changelog

### Перед коммитом
- [ ] `git diff --stat` — увидеть ВСЕ изменённые файлы
- [ ] `git diff` — понять суть каждого изменения
- [ ] Коммит-сообщение покрывает ВСЕ изменения, не только текущую сессию
