
Когда пользователь поправляет мою ошибку или указывает на неправильный паттерн поведения:

1. Признай ошибку.
2. Исправь текущую проблему.
3. **Обязательно спроси**: «Хотите, я обновлю свои инструкции (workflow / rules / skill), чтобы в будущем так не делать?»
4. Если пользователь согласен — обнови конкретный файл в `.agent/rules/`, `.agent/workflows/` или `.agent/skills/`, чтобы правило стало постоянным.
5. Не давай пустых обещаний вроде «в будущем буду делать X» — если это важно, запиши в файл.

## АВТОМАТИЧЕСКОЕ самообучение (без напоминаний!)

**После КАЖДОГО исправленного бага или решённой проблемы — СРАЗУ проверь:**

- Есть ли скилл/правило, которое описывает эту тему? → Обнови его с учётом нового знания.
- Нет скилла, но паттерн может повториться? → Создай новый скилл или правило.
- Примеры в скилле устарели или содержат антипаттерн? → Исправь примеры.

**НЕ ЖДАТЬ** пока пользователь попросит обновить — делать это проактивно перед `/save`.

### Триггеры для автоматического самообучения

| Ситуация | Действие |
|----------|----------|
| Исправил баг, связанный с существующим скиллом | Обнови скилл новым знанием |
| Исправил баг, нет связанного скилла | Предложи создать скилл/правило |
| Нашёл антипаттерн в примерах кода скилла | Исправь примеры |
| Пользователь указал на ошибку поведения | Обнови правило или workflow |
| Неполный комментарий к коммиту | Обнови workflow `/save` |
| Забыл проверить что-то перед действием | Добавь checklist-шаг |
| Использовал неправильный подход | Задокументируй правильный паттерн |
| Повторил одну и ту же ошибку дважды | Создай rule с явным запретом |
